## Codebase Patterns
- Freezed models live in `features/<name>/data/models/` with `part` files for `.freezed.dart` and `.g.dart`
- Repositories follow `features/<name>/data/repositories/` pattern with `Provider<Repo>` at bottom
- Firestore Timestamp → DateTime conversion done in `fromFirestore` extension methods
- Routes defined in `AppRoutes` class in `app_constants.dart`, wired in `app_router.dart`
- Screens use `ConsumerWidget` / `ConsumerStatefulWidget` for Riverpod access
- DiscoverScreen uses `DiscoverSegment` enum with embedded views (`embedded: true` flag)
- Provider directory category filter uses `StateProvider.autoDispose<String?>` pattern
- Always put `/create` routes before `/:id` routes in GoRouter to avoid wildcard matches
- Pre-existing test failures (4): profile_chat_nav_test EditProfileScreen tests + task_repository_test watchMyPostedTasks
