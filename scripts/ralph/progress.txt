## Codebase Patterns
- FakeCollectionReference.where() delegates to a shared FakeQuery — when chaining multiple .where() calls, the first is on the collection and the rest on FakeQuery. Both must track state.
- currentUserProvider is a StreamProvider<UserModel?> — use `ref.read(currentUserProvider.future)` to await the first emission instead of `.valueOrNull` with a raw fallback.
- Empty catch blocks should use `debugPrint` for best-effort operations (storage deletion, Firestore lookups) rather than silently swallowing errors.
- To conditionally render multiple slivers from an AsyncValue, use `...asyncValue.when(data: (x) => [sliver1, sliver2])` with spread.
- TaskModel extension has `isOpen`, `isAssigned`, `isCompleted` getters — prefer these over raw enum comparison.
