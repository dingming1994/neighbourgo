## Codebase Patterns
- Freezed model `toJson()` returns nested Dart objects (not raw maps) â€” use `jsonDecode(jsonEncode(model.toJson()))` for true fromJson/toJson roundtrip tests
- Test files for models go in `test/models/`, one file per model: `user_model_test.dart`, `task_model_test.dart`, etc.
- Use `timeago` package (already imported in task_list_screen.dart) for human-readable timestamps
- `AppColors`, `AppRadius`, `AppSpacing` are the design token classes in `lib/core/theme/app_theme.dart`
- `AppRoutes` constants are in `lib/core/constants/app_constants.dart`
- `AppCategories.all` is a `List<ServiceCategory>` with id, label, emoji, color fields; use `AppCategories.getById(id)` for safe lookup
- Providers use Riverpod: `StreamProvider`, `StateNotifierProvider`, `Provider` patterns
- `currentUserProvider` gives `AsyncValue<UserModel?>` via stream from Firestore
- `taskRepositoryProvider` exposes `watchMyPostedTasks(uid)` (all poster tasks) and `watchOpenTasks()` (open tasks)
- `TaskModel.budgetDisplay` extension gives formatted "S$Xâ€“S$Y" budget string
- Use `context.push()` for sub-routes, `context.go()` for main navigation tabs
- `flutter analyze` passes with no errors (warnings/infos from pre-existing code are OK)
- `withOpacity` is deprecated in this Flutter version; `.withValues(alpha: x)` is preferred, but existing code uses `withOpacity` throughout
- Use `extends Fake implements FirebaseAuth/FirebaseFirestore` for manual Firebase fakes in tests (no mockito needed)
- Firestore fakes need full chain: FakeFirebaseFirestore â†’ FakeCollectionReference â†’ FakeDocumentReference â†’ FakeDocumentSnapshot
- AuthRepository constructor accepts optional `auth:` and `db:` params â€” pass fakes directly for unit tests
- For PhoneAuthNotifier tests, extend AuthRepository with FakeAuthRepository that overrides methods (pass no-op fakes to super constructor)
- `PhoneAuthProvider.credential()` is pure Dart â€” works in tests without Firebase initialization
- Firestore query-chain fakes need: FakeCollectionReference â†’ FakeQuery (with where/orderBy/limit/snapshots) â†’ FakeQuerySnapshot â†’ FakeQueryDocumentSnapshot
- `UploadTask` in firebase_storage has private constructor â€” cannot be faked directly; test photo upload logic via doc verification or repository subclass
- TaskRepository constructor accepts optional `db:` and `storage:` params â€” pass fakes directly for unit tests
- For StateNotifier tests, use FakeRepository that overrides stream methods with StreamController.broadcast() for emit control
- `await Future<void>.delayed(Duration.zero)` is needed to let stream events propagate in StateNotifier tests
- For subcollection testing, FakeDocumentReference needs `.collection()` method returning FakeCollectionReference
- Batch operations need FakeWriteBatch that records updates list and applies them on `.commit()`
- BidRepository uses subcollections `tasks/{taskId}/bids/{bidId}` â€” constructor accepts optional `db:` param
- ChatRepository constructor requires both `db:` and `storage:` fakes â€” `FirebaseStorage.instance` triggers Firebase init error if not faked
- For document-level streams (e.g., getChatStream), FakeDocumentReference needs `.snapshots()` returning `Stream<DocumentSnapshot>`
- ProfileRepository constructor accepts optional `db:` and `storage:` params â€” same pattern as ChatRepository
- FakeQuery needs `limit()` method for queries that use `.limit()` (e.g., watchReviews)
- `updateShowcase` reads then writes â€” needs existing user doc data to be set with `exists: true` and proper data for `fromFirestore` to work
- Reviews subcollection path: `users/{uid}/reviews/{reviewId}` â€” use FakeDocumentReference.setSubcollection()
- Category labels are full names: 'Home Cleaning' (not 'Cleaning'), 'Tuition' (not 'Tutoring') â€” use `AppCategories.all[i].label` instead of hardcoding
- To test loading state in stream-based screens, use a repo that returns `StreamController<T>().stream` (never emits)
- Widget tests need `ProviderScope` with `overrides:` for Riverpod screens; use `buildTestWidget(child, overrides: [...])` helper
- Widget tests for layout-heavy screens need `tester.binding.setSurfaceSize(Size(480, 1200))` + overflow error suppression via `FlutterError.onError`
- Override `phoneAuthProvider` with `phoneAuthProvider.overrideWith((_) => FakeNotifier(state))` for StateNotifierProvider
- Override `currentUserProvider` with `currentUserProvider.overrideWith((_) => Stream.value(user))` for StreamProvider
- Override repository providers with `.overrideWithValue(fakeRepo)` for simple Provider<T>
